Pre-Setup & Access Provisioning

Setup
1.1
TOOL · Microsoft Admin Center + Entra ID
Confirm Business Premium Licenses & Required Add-Ons
Verify every PM and contributor has the right license — including mandatory add-ons — before touching any tool.
▾
  • Confirm all PMs have M365 Business Premium licenses in Admin Center (admin.microsoft.com) — max 300 users on this plan
  • Purchase and assign Power Apps per-user plan (~$20/user/month) for all builders and intake users — this is mandatory to unlock full Dataverse; do not skip
  • Purchase and assign Power BI Pro add-on (~$10/user/month) for all report consumers — not included in Business Premium
  • Confirm Microsoft Fabric capacity (F2+ SKU) is purchased and assigned to your workspace — separate Azure purchase
  • Assign Power Platform Environment Maker role to the PM or designated builder
  • Enable full Dataverse (standalone, not Teams) for the Power Platform environment — only available after Power Apps per-user plan is assigned
  • Ensure MFA + Conditional Access is active via Entra ID P1 — pair with Intune device compliance policies
⚠️

Do NOT build on Dataverse for Teams. It cannot be linked to Fabric, has a 2 GB storage cap, and does not support the security role model required here. Confirm the environment type shows "Production" or "Sandbox" — not "Teams" — in the Power Platform Admin Center before proceeding.

💡

Document your environment name and region. Every project reuses the same Power Platform environment — don't create a new one per project.

1.2
TOOL · Microsoft Fabric Workspace
Create the Fabric Workspace (One per Project)
Set up a dedicated Fabric workspace as the data backend for each project.
▾
  • Go to app.fabric.microsoft.com → Workspaces → New Workspace
  • Name convention: PRJ-[ProjectCode]-Workspace
  • Assign Fabric capacity to the workspace (F2+ SKU — not Trial)
  • Create a Lakehouse inside: PRJ-[ProjectCode]-Lakehouse
  • Add PM and data team as Workspace Members or Contributors
01

Configure Device Compliance (Intune)  ✦ BP Exclusive

Security
1.3
TOOL · Microsoft Intune + Entra ID P1
Enforce Compliant-Device Access to the Intake Platform
Leverage Intune — included exclusively in Business Premium — to gate Power Apps and Teams access to compliant devices only.
▾
  • In intune.microsoft.com → Devices → Compliance Policies → Create Policy
  • Require: BitLocker enabled · OS minimum version · Defender for Business reporting healthy
  • In Entra ID → Conditional Access → New Policy → target Power Apps and Teams → require device compliance
  • Assign Intune app protection policies to the Power Apps mobile app for field crew devices
  • Enroll all PM and contributor devices in Intune before go-live; verify non-compliant devices are blocked
✦

This step is exclusive to Business Premium — E3 does not include Intune. It adds meaningful endpoint security depth for Ontario One Call's field-facing workflows at no extra licensing cost.

02

Build the Intake UI in Power Apps

Build
2.1
TOOL · Power Apps Studio (make.powerapps.com)
Create the Canvas App from Template
Build the intake form as a Canvas App — the reusable UI template for all projects.
▾
  • Confirm you are working in the full Dataverse environment — check the environment picker top-right; it must NOT read "Teams"
  • Go to make.powerapps.com → + Create → Canvas App from blank
  • Name the master template: IntakeForm-MASTER-Template
  • Build a multi-screen wizard: Screen 1 = Project Info · Screen 2 = Financial Details · Screen 3 = Review & Submit
  • Add a project selector dropdown — one app serves multiple projects via varProjectCode
  • Set field-level conditional visibility on financial fields
💡

For each new project, add a new option to the project dropdown. One app, many projects — no duplication.

2.2
TOOL · Power Apps → Dataverse Connection
Connect App to Dataverse & Publish to Teams
Wire form fields to Dataverse and surface the app in Teams via SSO.
▾
  • Data → Add data → Dataverse → select your intake table
  • Submit button: use Patch(IntakeRequests, Defaults(IntakeRequests), {...})
  • Always include ProjectCode as mandatory in every Patch call
  • Publish → Add to Teams → pin via Teams Admin Center Setup Policies
  • Add as a Tab in each project Teams channel · Test SSO (zero additional prompts)

04

Enable Dataverse → Fabric Link

Fabric
4.1
TOOL · Microsoft Fabric · Link to Dataverse
Zero-Copy Link: Dataverse → OneLake
Connect Dataverse directly to Fabric — no ETL, no pipeline, near real-time.
▾
  • Confirm the Dataverse environment type is standalone (full) — Power Platform Admin Center → Environments → Type must read "Production" or "Sandbox", not "Teams"
  • In Fabric Lakehouse → Get Data → Dataverse → select your environment
  • Select the intake request table → enable "Link to Microsoft Fabric"
  • Confirm the table shortcut appears in OneLake → Tables
  • Run a Fabric Notebook query to validate data is flowing correctly
  • Create per-project SQL views in the Lakehouse SQL endpoint: vw_PRJ[Code]_Intake
✅

The link is near-real-time. Data submitted in Power Apps is queryable in Fabric within seconds — no scheduled refresh needed.

⚠️

If this step fails with an environment type error, the Dataverse environment is still a Teams environment. Return to Phase 1 Step 1.1 — provision a full Power Platform environment with the Power Apps per-user plan assigned first.

05

Automate Approvals & Notifications

Automate
5.1
TOOL · Power Automate + Teams Adaptive Cards
Build the Approval Workflow Template
One reusable flow handles approvals for all projects — approver pulled dynamically from the record.
▾
  • Trigger: "When a row is added/modified" → Dataverse → IntakeRequests · condition: Status = 'Submitted'
  • Start and wait for an approval → Approver = cr_approver from the record
  • On approval: update cr_status = 'Approved' + send Teams Adaptive Card to submitter
  • On rejection: update cr_status = 'Rejected' + Teams notification with reason
  • Save as Solution-aware flow for portability across environments

Scale — New Project Spin-Up

7-Step New Project Onboarding Checklist
No new infrastructure. A new project is just configuration.
▾
  • 1 — Register: Add ProjectCode to Power Apps dropdown lookup table
  • 2 — Security: Create Entra ID group SG-PRJ[Code]-Members, assign users, confirm all devices are enrolled in Intune
